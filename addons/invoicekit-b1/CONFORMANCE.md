# InvoiceKit SAP Business One Connector Conformance

This package implements the T-1502 runbook contract from
`docs/operators/ERP-CONNECTORS.md`.

| Requirement | Evidence |
| --- | --- |
| SAP B1 add-on source lives under `addons/invoicekit-b1/` | `manifest.json`, `src/*.cs`, and `src/*.csproj` |
| Add-on registration data is represented as generator input | `package/addon-registration.input.xml` |
| Add-on exposes an ERP-native Send via InvoiceKit command | `src/InvoiceKitApplication.cs` |
| Connector posts to the sidecar `/v1/transmit` endpoint | `src/InvoiceKitSidecarClient.cs` |
| Transmission payload carries trace, tenant, idempotency, and gateway attempt identifiers | `src/InvoiceKitSidecarClient.cs` |
| Static conformance checks run on every PR | `.github/workflows/sap-b1-connector.yml` |
| SAP B1 SDK package gate is wired conditionally | `.github/workflows/sap-b1-connector.yml` |
| Uninstall path does not corrupt host ERP data | `connectors/sap-b1/README.md` |

SAP's `.ard` registration file is generated by SAP Business One SDK tools from
the add-on setup and registration input. The checked-in package records the
source inputs and CI gate; the signed/generated `.ard` artifact requires an
operator-provisioned SAP Business One SDK installation.
