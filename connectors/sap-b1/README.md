# connectors/sap-b1

SAP Business One connector for InvoiceKit.

The implementation lives in `addons/invoicekit-b1/` as a SAP Business One
UI API add-on. It adds a `Send via InvoiceKit` menu command for A/R invoices,
extracts the active invoice fields from the SAP B1 client, and posts invoice
JSON to the InvoiceKit sidecar `/v1/transmit` endpoint.

## Local checks

Run the static package conformance checks from the repository root:

```bash
python3 -m pytest addons/invoicekit-b1/tests -q
```

The full SAP SDK build is wired in `.github/workflows/sap-b1-connector.yml`.
It is conditional because SAP Business One SDK libraries and the
`AddOnRegDataGen.exe` tool must come from an operator-provisioned SAP B1 SDK
installation.

## Package layout

- `addons/invoicekit-b1/manifest.json` - add-on metadata and sidecar contract.
- `addons/invoicekit-b1/package/addon-registration.input.xml` - source input
  for SAP's add-on registration data generator.
- `addons/invoicekit-b1/src/InvoiceKit.B1Addon.csproj` - .NET Framework add-on
  project with SAP UI API and DI API references.
- `addons/invoicekit-b1/src/InvoiceKitApplication.cs` - SAP B1 menu command and
  event wiring.
- `addons/invoicekit-b1/src/InvoiceKitSidecarClient.cs` - sidecar HTTP client.

## Install and uninstall behavior

The generated SAP add-on registration data installs a client-side add-on. The
connector does not alter SAP B1 database tables directly. It reads the active
A/R invoice, sends it to the sidecar, and reports the submission result through
the SAP B1 status bar. Uninstalling the add-on removes the menu command and
client executable while leaving native SAP B1 invoice data unchanged.

SAP's `.ard` file is generated by the SAP Business One SDK registration tooling
from the add-on executable and registration input. The generated artifact is
environment-specific and is produced in the operator SAP SDK environment rather
than checked in here.
